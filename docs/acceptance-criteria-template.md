# Acceptance Criteria &amp; Definition of Done (Template)

Use this template when creating issues or user stories so acceptance criteria are concise, testable, and aligned to the Definition of Done (DoD).

Issue / Story: [title]

Goal
- Short statement of the user or business outcome.

Acceptance criteria (must be verifiable):
- Given [context], when [action], then [expected result]
- ...

Non-functional requirements:
- Performance, accessibility, security, localization, etc.

Definition of Done (check all that apply before merge):
- [ ] Code reviewed by at least one peer
- [ ] Unit and integration tests added/updated
- [ ] Feature toggles or rollout plan documented (if needed)
- [ ] CI green
- [ ] Documentation updated (user docs or internal runbook)
- [ ] QA verification completed

Notes
- Link to designs, mockups, test plans, and relevant issues.
