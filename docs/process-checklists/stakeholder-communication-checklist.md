# Stakeholder Communication Checklist

Use this checklist when preparing stakeholder updates to ensure clear, effective communication.

## Planning Phase

- [ ] **Identify Audience**: List all stakeholders who need to receive this communication (executives, customers, team members, etc.)
- [ ] **Define Objective**: Clarify the purpose of the communication (status update, decision request, risk notification, etc.)
- [ ] **Key Messages**: Outline the 3-5 most important points stakeholders need to know

## Preparation Phase

- [ ] **Required Approvals**: Identify who needs to review and approve the communication before it is sent
- [ ] **Communication Channel**: Select the appropriate channel (email, meeting, demo, status report, etc.)
- [ ] **Supporting Materials**: Prepare any necessary attachments, slides, or documentation

## Delivery Phase

- [ ] **Send Communication**: Deliver the message through the chosen channel at the appropriate time
- [ ] **Feedback Loop**: Provide a clear way for stakeholders to ask questions or provide feedback
- [ ] **Document Responses**: Record stakeholder questions, concerns, and feedback for follow-up

## Follow-Up Phase

- [ ] **Follow-Up Actions**: Track and complete any action items or commitments made during the communication
- [ ] **Confirmation**: Verify that key stakeholders received and understood the message
- [ ] **Archive**: Store the communication and responses for future reference
